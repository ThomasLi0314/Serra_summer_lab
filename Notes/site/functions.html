Functions Reference

Every LCStool function, grouped by role. Each card includes the signature, key options, the underlying formula, and where it sits in the pipeline.

1. Grid & flow integration

initialize_ic_grid util

Build a Cartesian grid of initial conditions.
Signature
position = initialize_ic_grid(resolution, domain)
Math
Produces the tensor product grid $\{(x_i, y_j)\}$ with $x_i = x_{\min} + \tfrac{i-1}{N_x-1}(x_{\max}-x_{\min})$, $y_j = y_{\min} + \tfrac{j-1}{N_y-1}(y_{\max}-y_{\min})$, flattened meshgrid-style.
Output
position — $N \times 2$ with $N = N_x N_y$, one $(x,y)$ per row.
Used by
Nearly every downstream routine that needs uniform seeds.

equal_resolution util

Pick $N_y$ such that $\Delta x \approx \Delta y$.
Signature
[resolutionY, deltaX] = equal_resolution(domain, resolutionX)
Math
$\Delta x = \tfrac{x_{\max}-x_{\min}}{N_x-1}$, then $N_y = \text{round}\!\left(\tfrac{y_{\max}-y_{\min}}{\Delta x}\right) + 1$.
Why
The auxiliary-grid Cauchy-Green computation assumes isotropic cells; unequal spacing triggers a warning.

integrate_flow util

Advect a cloud of particles with ode45.
Signature
flowSolution = integrate_flow(flow, initialPosition, useEoV, verbose)
Flow struct
Fields used: derivative, timespan, coupledIntegration (default true), odeSolverOptions, cgStrainMethod.name.
Modes
Coupled: one ode45 on the stacked state of all particles. Fastest for large grids but memory-heavy.
Uncoupled: one ode45 per particle, parallelised with parfor. Memory-friendly.
EoV
If useEoV = true, the initial state for each particle is augmented with an identity Jacobian flat-packed to 4 entries, and the ODE integrates position + flow-map gradient together (see Math §1).

apply_periodic_bc util

Wrap positions across periodic axes.
Signature
[posPeriodic, wrapIdx] = apply_periodic_bc(position, periodicBc, domain)
Formula
For axis $m$ with $\text{periodicBc}(m) = \text{true}$: $\;\; p_{\text{new}}(m) = \text{mod}\!\bigl(p(m) - \text{dom}_{\min,m},\, L_m\bigr) + \text{dom}_{\min,m}$.
Limitation
Only the x-axis is implemented; a y-periodic request is fatal.

2. Cauchy-Green strain & FTLE

eig_cgStrain core

The workhorse. Computes eigenvalues and eigenvectors of $C = \nabla F^\top \nabla F$ on the whole grid.
Signature
[cgV, cgD] = eig_cgStrain(derivative, domain, resolution, timespan, …)
Key options
'method' : 'finiteDifference' (default) or 'equationOfVariation'.
'auxGridRelDelta' (default $10^{-2}$) — spacing of auxiliary grid used to compute $\nabla F$.
'eigenvalueFromMainGrid' (default true) — compute eigenvalues on the main grid, eigenvectors on the auxiliary grid.
'incompressible' — force $\lambda_1\cdot\lambda_2 = 1$.
'odeSolverOptions' — odeset struct.
Math (finite-difference mode)
For every main-grid point $\mathbf{x}_0$, 4 auxiliary seeds at $\mathbf{x}_0 \pm \delta \hat{e}_x$ and $\mathbf{x}_0 \pm \delta \hat{e}_y$ are advected. Then $$ \nabla F_{ij} \approx \frac{F_i(\mathbf{x}_0 + \delta \hat e_j) - F_i(\mathbf{x}_0 - \delta \hat e_j)}{2\delta}. $$ Finally $C_{11} = F_{11}^2 + F_{21}^2$, $C_{12} = F_{11}F_{12} + F_{21}F_{22}$, $C_{22} = F_{12}^2 + F_{22}^2$; its $2\times 2$ spectrum is analytic.
Internals
ode45_vector — coupled ODE advection of all seeds at once.
auxiliary_position — build the 8-column auxiliary grid.
compute_cgStrain — assemble $C$ from final positions.
eig_array — $2\times 2$ symmetric eigen-solver (arrayfun-friendly).
negative_to_nan — tag non-positive-definite points.
Output layout
cgD — $N\times 2$: $[\lambda_1, \lambda_2]$ with $\lambda_1 \le \lambda_2$.
cgV — $N\times 4$: $[\xi_{1x}, \xi_{1y}, \xi_{2x}, \xi_{2y}]$.

ftle util

Finite-Time Lyapunov Exponent from the largest Cauchy-Green eigenvalue.
Signature
ftleField = ftle(max_eigenvalue, timespan)
Formula
$\Lambda_{t_0}^{t}(\mathbf{x}_0) = \dfrac{1}{2\,|t-t_0|}\,\ln \lambda_2(\mathbf{x}_0)$
Usage
ftleField = ftle(cgD(:,2), diff(timespan));

3. Hyperbolic LCS — shrink & stretch lines

seed_curves_from_lambda_max hyperbolic

Place curves at local extrema of an eigenvalue field and integrate the matching eigenvector line.
Signature
[curves, seeds] = seed_curves_from_lambda_max(distance, curveMaxLength, cgEigenvalue, cgEigenvector, flowDomain, flowResolution, …)
Arguments
distance — minimum separation between seeds (prevents curve overlap).
curveMaxLength — arc-length cap per direction.
cgEigenvalue — pass cgD(:,2) for shrink, -cgD(:,1) for stretch.
cgEigenvector — matching pair: cgV(:,1:2) for $\xi_1$ or cgV(:,3:4) for $\xi_2$.
'periodicBc', 'nMaxCurves', 'odeSolverOptions'.
Algorithm
(1) find all local maxima on the grid via local_max2D_gridded with a circular-mask distance threshold;
(2) sort descending; for each maximum not yet "covered" by previous curves,
(3) integrate the eigenvector field forward and backward from the seed with integrate_line;
(4) concatenate the two halves;
(5) flag a circular neighbourhood around every point on the new curve so later seeds cannot start there.
Edge handling
If the seed lies on the domain boundary, the code checks the eigenvector sign there and skips the integration direction that would immediately leave the domain.
Theory link
Implements conditions (2) + (3) of the repelling-LCS theorem (Math §4).

integrate_line core

Generic integrator for non-orientable vector fields. The engine behind every strain-line and λ-line integration.
Signature
[position, iEvent, largeAngle] = integrate_line(timespan, initCond, domain, resolution, periodicBc, vectorGrid, odeOpts, [poincareSection], 'checkDiscontinuity', true/false)
Non-orientability handling
Uses a valueHandle to carry the previous vector direction into the ODE output function. At each step, $$ \mathbf{v}_{\text{new}} \leftarrow \operatorname{sign}(\mathbf{v}_{\text{prev}}\cdot \mathbf{v}_{\text{new}})\,\mathbf{v}_{\text{new}}. $$ This keeps the integrated curve coherent even though eigenvectors only have directional (not vector) meaning.
Discontinuity detection
At each position, the four vectors at the enclosing cell corners are compared. If any pair subtends an angle > π/2, the cell is flagged and the interpolant is rebuilt with locally sign-flipped neighbours, yielding a continuous interpolant for that cell.
Event types (returned as iEvent)
0: no termination (hit timespan(2))
1: crossed the Poincaré section (closed orbit candidate)
2: left the domain
3: hit NaN in the vector field

remove_strain_in_elliptic hyperbolic

Trim shrink/stretch lines that enter an elliptic LCS boundary.
Signature
strainlinePosition = remove_strain_in_elliptic(strainlinePosition, ellipticLcs)
Algorithm
For each strainline, the points inside the elliptic boundary (via inpolygon) are identified. The curve is split around the first and last "inside" points; the two outside segments are kept, the inside portion discarded.
Caveat
Assumes at most one crossing per strainline (flagged in source). If a strainline enters and exits the elliptic region multiple times, pieces may be lost.

4. Elliptic LCS — λ-lines & Poincaré orbits

lambda_line elliptic

Build the generalised $\eta_\lambda^\pm$ vector fields, pointwise.
Signature
[etaPos, etaNeg] = lambda_line(cgV, cgD, lambda, 'forceComplexNaN', forceComplexNaN)
Formula (Haller & Beron-Vera 2013, Eq. 3.2)
$ \eta_\lambda^\pm = \sqrt{\dfrac{\lambda_2-\lambda^2}{\lambda_2-\lambda_1}}\,\xi_1 \pm \sqrt{\dfrac{\lambda^2-\lambda_1}{\lambda_2-\lambda_1}}\,\xi_2 $
Outputs
etaPos, etaNeg — each $N\times 2$.
Option
'forceComplexNaN': when $\lambda^2$ lies outside $[\lambda_1, \lambda_2]$ the square roots become imaginary; setting this to true replaces those points with NaN.

poincare_closed_orbit elliptic

Find closed orbits of a single η-field through a single Poincaré section.
Signature
[closedOrbits, orbits, hFig] = poincare_closed_orbit(domain, resolution, vectorField, poincareSection, …)
Poincaré-section struct
endPosition: $2\times 2$, rows are the two endpoints.
numPoints: how many seeds along the section.
integrationLength: [0, orbitMaxLength] cap on integration time.
Key options
'nBisection' (default 5) — root-finding refinement steps.
'dThresh' (default $10^{-2}$) — reject zero-crossings whose neighbours have return distance larger than this fraction of section length.
'showGraph' — plot the Poincaré return-distance map.
Algorithm summary
(i) seed, integrate until back to $\Sigma$; (ii) rotate to align $\Sigma$ with x-axis; (iii) find zero-crossings of "return minus initial" coordinate; (iv) refine with bisection; (v) retain roots passing the neighbour-distance filter; (vi) integrate and sort orbits innermost → outermost.

poincare_closed_orbit_multi elliptic

Loop poincare_closed_orbit over multiple sections and both η-fields.
Signature
[closedOrbits, orbits] = poincare_closed_orbit_multi(domain, resolution, etaPos, etaNeg, PSList, …)
Output layout
closedOrbits{m}{1}: list of closed orbits (innermost → outermost) around section $m$ via $\eta^+$.
closedOrbits{m}{2}: same via $\eta^-$.

poincare_closed_orbit_range elliptic

The outermost loop — sweep a range of λ values.
Signature
[clPos, clNeg] = poincare_closed_orbit_range(domain, resolution, cgV, cgD, lambda, PSList, …)
Output layout
Cell arrays of size [nLambda, nPoincareSection]. Element {iLambda, iPs} holds the closed-orbit list returned by poincare_closed_orbit_multi.

elliptic_lcs elliptic

Final selection: pick the largest-area outermost orbit per Poincaré section.
Signature
ellipticLcs = elliptic_lcs(closedLambdaLine)
Algorithm
For each Poincaré section, iterate over all λ values, compute polyarea of the outermost closed orbit, and retain the one with maximum area. Sections with no closed orbit at any λ are dropped.

5. Diagnostics & utilities

eig_error diagnostic

Residual of the eigenvalue problem.
Signature
e = eig_error(A, V, D)
Formula
$e_i = \|(A - D_{ii} I)\,v_i\|$

cgStrain_stats diagnostic

Printed summary of a Cauchy-Green computation.
Reports
• Number of negative eigenvalues (should be 0 — indicates integration error).
• Number of NaN values.
• Min/max/mean/median of $\|A v - \lambda v\|$ (Eigensystem residual).
• Min/max/mean/median of $\lambda_1 \lambda_2$ (should be 1 for incompressible flows).
• $1 - \max \lambda_1$ and mean $|\det C - 1|$.

angle_change diagnostic

Angular change of a vector field between neighbours.
Signature
[thetaX, thetaY, thetaMax] = angle_change(vector)
Formula
$ \theta = \arccos\!\bigl(\tfrac{\mathbf{a}\cdot\mathbf{b}}{\|\mathbf{a}\|\|\mathbf{b}\|}\bigr) $ between cells $(i,j)$ and $(i+1,j)$ or $(i,j+1)$.
Use
Feed the result to print_theta_hist for a histogram, or plot $\theta_{\max}$ as a 2-D field to spot discontinuity patches.

print_theta_hist diagnostic

Tabulate a histogram of angle changes in 10°-wide bins from 0° to 180°.

valueHandle util

One-property handle class used to pass mutable state through ode45 output/event callbacks.
Why it exists
MATLAB anonymous functions capture workspace values by copy. To track "the previous vector direction" across ODE callbacks, integrate_line needs a handle; rather than depending on global state, LCStool uses this tiny class.
