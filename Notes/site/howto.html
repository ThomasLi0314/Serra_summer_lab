How-To Guide

Apply LCStool to your own velocity field — from the first derivative wrapper to a complete driver.

1. Input conventions you must obey

LCStool is strict about how the velocity is supplied. Get this wrong and your eig_cgStrain call will produce silent garbage.

The derivative function signature

Three arguments, in this order:

  1. t — scalar time.
  2. x — column vector of stacked positions.
  3. useEoV — logical flag; set to false when wrapping unless you know you need the equation of variation.

Stacking format

If you are integrating $N$ particles, the input is

$$ \mathbf{x}_{\text{in}} = [x_1,\, y_1,\, x_2,\, y_2,\, \ldots,\, x_N,\, y_N]^\top \in \mathbb{R}^{2N}. $$

and you must return a column of the same length:

$$ \mathbf{v}_{\text{out}} = [u_1,\, v_1,\, u_2,\, v_2,\, \ldots,\, u_N,\, v_N]^\top \in \mathbb{R}^{2N}. $$

In MATLAB, the standard index pattern is:

idxX = 1:2:numel(x)-1;  % odd indices  — particle x-positions
idxY = 2:2:numel(x);    % even indices — particle y-positions

dydt = nan(size(x));
dydt(idxX) = u(t, x(idxX), x(idxY));
dydt(idxY) = v(t, x(idxX), x(idxY));

Equation-of-variation extension (optional)

If you want 'method','equationOfVariation' in eig_cgStrain, each particle carries 4 additional states ($\Phi_{11}, \Phi_{12}, \Phi_{21}, \Phi_{22}$). The stride becomes 6 instead of 2; the extra entries satisfy $\dot\Phi = \nabla\mathbf{v}\,\Phi$. You can copy the pattern from demo/double_gyre/derivative.m — it is about a dozen lines.

In practice, the default finite-difference method works for virtually all use cases and does not require you to code the EoV block.

2. Workflow for an analytical velocity field

Scenario: you have equations for $\mathbf{v}(t, \mathbf{x})$ — e.g., a stream function or a system of ODEs — and want to find LCSs on some time interval.

Step 1 — Write the derivative

Create my_derivative.m:

function dydt = my_derivative(t, x, useEoV)
    idxX = 1:2:numel(x)-1;
    idxY = 2:2:numel(x);
    dydt = nan(size(x));

    % --- replace with your velocity ---
    dydt(idxX) =  sin(x(idxY)) + 0.1*cos(t);
    dydt(idxY) = -cos(x(idxX));
    % ----------------------------------

    if useEoV
        error('my_derivative:noEoV', 'EoV not implemented; call with useEoV=false');
    end
end

Step 2 — Complete driver template

%% 0. Add LCStool to the path
addpath('path/to/LCStool-master/LCStool-1-0');

%% 1. Domain, grid, time
domain      = [0, 2*pi; 0, 2*pi];
timespan    = [0, 20];
resolutionX = 400;
[resolutionY, deltaX] = equal_resolution(domain, resolutionX);
resolution  = [resolutionX, resolutionY];

%% 2. Velocity wrapper
lDerivative   = @(t,x,~) my_derivative(t, x, false);
incompressible = true;   % set false if flow is not area-preserving

%% 3. Cauchy-Green eigen-decomposition
cgOde = odeset('relTol', 1e-5);
[cgV, cgD] = eig_cgStrain(lDerivative, domain, resolution, timespan, ...
                'incompressible', incompressible, ...
                'odeSolverOptions', cgOde);

%% 4. (Optional) Quick FTLE sanity check
ftleField = ftle(cgD(:,2), diff(timespan));
hF = setup_figure(domain);
plot_ftle(hF, domain, resolution, ftleField);
title(hF, 'FTLE — look for ridges');
drawnow;

%% 5. Hyperbolic LCSs
shrinkOde = odeset('relTol', 1e-6);

shrinkLine = seed_curves_from_lambda_max( ...
     2*deltaX, 20, cgD(:,2), cgV(:,1:2), domain, resolution, ...
     'odeSolverOptions', shrinkOde);

stretchLine = seed_curves_from_lambda_max( ...
    10*deltaX, 20, -cgD(:,1), cgV(:,3:4), domain, resolution, ...
    'odeSolverOptions', shrinkOde);

%% 6. Elliptic LCSs — pick Poincaré sections from the FTLE plot
poincareSection = struct('endPosition',{},'numPoints',{},'orbitMaxLength',{});
poincareSection(1).endPosition = [pi, pi; pi+1, pi+1];   % EDIT: through vortex core
[poincareSection.numPoints]    = deal(100);
for i = 1:numel(poincareSection)
    r = hypot(diff(poincareSection(i).endPosition(:,1)), ...
              diff(poincareSection(i).endPosition(:,2)));
    poincareSection(i).orbitMaxLength = 2*(2*pi*r);
end
lambda = 0.9 : 0.02 : 1.1;

[clPos, clNeg] = poincare_closed_orbit_range(domain, resolution, cgV, cgD, ...
                    lambda, poincareSection, 'forceEtaComplexNaN', true);
ellipticLcs = [elliptic_lcs(clPos), elliptic_lcs(clNeg)];

%% 7. Trim hyperbolic curves that enter elliptic regions
for i = 1:numel(ellipticLcs)
    shrinkLine  = remove_strain_in_elliptic(shrinkLine,  ellipticLcs{i});
    stretchLine = remove_strain_in_elliptic(stretchLine, ellipticLcs{i});
end

%% 8. Final plot
hAx = setup_figure(domain);
title(hAx, 'LCSs');
hEll = plot_elliptic_lcs(hAx, ellipticLcs);
set(hEll, 'color', [0 .6 0], 'linewidth', 2);
cellfun(@(p) plot(hAx, p(:,1), p(:,2), 'r'), shrinkLine);
cellfun(@(p) plot(hAx, p(:,1), p(:,2), 'b'), stretchLine);
uistack(hEll, 'top');
Tip. The only step that usually needs iteration is placing the Poincaré sections. Use the FTLE map from step 4 to eyeball where the vortex cores are, then place each section so it crosses exactly one vortex and is longer than the expected orbit diameter.

3. Workflow for gridded velocity data

Scenario: you have a CFD/DNS output, satellite altimetry, or model output as arrays on a regular spatio-temporal grid.

Step 1 — Build interpolants

Suppose you have variables

  • t — vector of nT time stamps.
  • xGrid, yGrid — vectors of nX and nY grid points.
  • U, V — 3-D arrays of size [nT, nY, nX] (note: Y before X!).
% Zero velocity at spatial boundaries to stop NaN leakage
U(:, [1,end], :) = 0;  U(:, :, [1,end]) = 0;
V(:, [1,end], :) = 0;  V(:, :, [1,end]) = 0;

U_int = griddedInterpolant({t, yGrid, xGrid}, U, 'spline');
V_int = griddedInterpolant({t, yGrid, xGrid}, V, 'spline');

Step 2 — Derivative wrapper

function dydt = my_deriv_data(t, pos, Uin, Vin)
    N    = numel(pos)/2;
    xP   = pos(1:2:end-1);
    yP   = pos(2:2:end);
    tCol = t*ones(N, 1);
    dydt = nan(2*N, 1);
    dydt(1:2:end-1) = Uin(tCol, yP, xP);     % note: (t, y, x) order
    dydt(2:2:end)   = Vin(tCol, yP, xP);
end

lDerivative = @(t,x,~) my_deriv_data(t, x, U_int, V_int);

The rest of the driver (stages 1–8 from §2) is identical. Only the derivative construction differs.

Checklist for gridded data

  • Units. If positions are in metres and velocities in m/s, keep timespan in seconds. Mixed units (common mistake) produce nonsense.
  • Zero-padding. The boundary layer of your velocity arrays must be zero; otherwise griddedInterpolant's 'spline' method extrapolates wildly at the edge.
  • Spatial resolution. Set resolutionX so $\Delta x$ is close to the native grid spacing of your data. Oversampling amplifies interpolation noise into $C$.
  • Longitude/latitude. For lat/lon data at high latitudes, project to a local flat plane first or embed metric factors in your derivative.
  • Eigenvalue source. On noisy data, set 'eigenvalueFromMainGrid', false — aux-grid eigenvalues are less noisy.

4. Recommended debugging workflow

Don't jump straight to the full LCS pipeline. Stage your work:

  1. Animate first. Build a minimal flow struct (see animate_flow) with a coarse grid (resolution = [20, 10]) and run animate_flow. Does the motion look right? If not, fix your derivative.
  2. FTLE next. Run eig_cgStrain at coarse resolution (e.g. resolutionX = 200) and plot FTLE with plot_ftle. Ridges should be continuous and visually plausible. If it looks pixelated or noisy, tighten odeSolverOptions (e.g. 'relTol', 1e-7) or increase resolution.
  3. Check stats. Run cgStrain_stats if it's available in your workflow:
    • Negative eigenvalues > 0 means your ODE tolerances are too loose.
    • Mean $|\det C - 1| \gg 10^{-3}$ on an incompressible flow means the same.
  4. Hyperbolic LCSs. Now run seed_curves_from_lambda_max. Start with nMaxCurves = 20 to keep the figure readable; increase later.
  5. Elliptic LCSs last. This is the fussiest step. Place Poincaré sections one-by-one through cores identified in step 2. Turn on 'showGraph', true in poincare_closed_orbit_range so you can see the return-distance curves and zero-crossings — this is by far the best debugging aid when no closed orbits are detected.
  6. Refine. Once the pipeline succeeds, increase resolution (e.g. resolutionX = 500–1000), tighten tolerances, and re-run for publication-quality figures.

Quick recipe card: "I just want FTLE"

lDerivative = @(t,x,~) my_derivative(t, x, false);
[~, cgD]    = eig_cgStrain(lDerivative, domain, resolution, timespan);
ftleField   = ftle(cgD(:,2), diff(timespan));
plot_ftle(setup_figure(domain), domain, resolution, ftleField);

Quick recipe card: "I just want elliptic LCSs"

[cgV, cgD] = eig_cgStrain(lDerivative, domain, resolution, timespan, 'incompressible', true);
[clPos, clNeg] = poincare_closed_orbit_range(domain, resolution, cgV, cgD, lambda, poincareSection, 'forceEtaComplexNaN', true);
ellipticLcs = [elliptic_lcs(clPos), elliptic_lcs(clNeg)];
hAx = setup_figure(domain);
plot_elliptic_lcs(hAx, ellipticLcs);
