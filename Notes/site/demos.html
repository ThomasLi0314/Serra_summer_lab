Demo Cases

Three canonical flows that ship with LCStool — their equations, their parameters, and what to expect.

Each demo folder contains three key files:

  • derivative.m — the velocity RHS.
  • elliptic_hyperbolic_lcs.m — the driver script that runs the full pipeline.
  • flow_animation.m — a sanity-check animation driver.

To run any demo from the MATLAB prompt:

cd(fullfile('demo', 'double_gyre'))   % or bickley_jet / ocean_dataset
addpath(fullfile('..', '..'))
elliptic_hyperbolic_lcs

1. Double Gyre textbook

Physical setup

The classic time-periodic double gyre of Shadden et al. (DOI:10.1016/j.physd.2005.10.007) — two counter-rotating cells on $[0,2]\times[0,1]$ whose dividing line oscillates sinusoidally:

$$ \psi(x,y,t) = A\,\sin\!\bigl(\pi f(x,t)\bigr)\,\sin(\pi y), $$ $$ f(x,t) = a(t)\,x^2 + b(t)\,x,\quad a(t) = \varepsilon\sin(\omega t),\ b(t) = 1 - 2\varepsilon\sin(\omega t). $$

The velocity follows from $\mathbf{v} = (-\partial_y \psi,\ \partial_x \psi)$:

$$ u = -\pi A\,\sin(\pi f)\,\cos(\pi y), \qquad v = \phantom{-}\pi A\,\cos(\pi f)\,\sin(\pi y)\,\bigl(2a(t)x + b(t)\bigr). $$

MATLAB implementation

Inside demo/double_gyre/derivative.m:

a = epsilon*sin(omega*t);
b = 1 - 2*epsilon*sin(omega*t);
forcing = a*x(idx1).^2 + b*x(idx1);

derivative_(idx1) = -pi*amplitude*sin(pi*forcing).*cos(pi*x(idx2));
derivative_(idx2) =  pi*amplitude*cos(pi*forcing).*sin(pi*x(idx2)).*(2*a*x(idx1) + b);

The idx1/idx2 dance is the stacking convention — odd indices hold $x$, even hold $y$. The equation-of-variation block below adds four more entries per particle with the 2×2 velocity gradient $\nabla \mathbf{v}$.

Expected output

  • Two green elliptic LCSs, one inside each gyre.
  • A dense web of red shrink lines (repelling) and blue stretch lines (attracting) sweeping across and between the gyres.
  • The central S-shaped repelling manifold is the most visually striking feature — it is the famous "transport barrier" between the two gyres.

Minimal animation driver

% demo/double_gyre/flow_animation.m
epsilon = .1; amplitude = .1; omega = pi/5;
flow.imposeIncompressibility = true;
flow.derivative = @(t,x,useEoV) derivative(t, x, useEoV, epsilon, amplitude, omega);
flow.domain     = [0,2; 0,1];
flow.timespan   = [0, 20];
flow.resolution = [2, 1]*10;
flow = animate_flow(flow);

2. Bickley Jet geophysical

Physical setup

An idealised zonal jet with three travelling Rossby-wave perturbations, on a zonally periodic channel (Beron-Vera et al., DOI:10.1016/j.physd.2012.06.012). Stream function:

$$ \psi(x,y,t) = U\,L_y\!\left[\tanh\!\bigl(y/L_y\bigr) - \operatorname{sech}^2\!\bigl(y/L_y\bigr)\,\Re\!\sum_{n=1}^{3} \varepsilon_n\, f_n(t)\, e^{i\,k_n x}\right] - c_3\,y, $$ $$ k_n = \tfrac{2\pi n}{L_x},\quad c_2 = 0.205\,U,\ c_3 = 0.461\,U,\quad \sigma_1 = \tfrac{1}{2} k_2 (c_2 - c_3),\ \sigma_2 = 2\sigma_1. $$

Three forcing cases are implemented:

  • Case 1 — time-periodic: $f_n(t) = e^{i \sigma_n t}$.
  • Case 2, 3 — time-aperiodic: $f_1(t) = f_2(t) = \alpha\,\phi_1\!\bigl(t/T_{BV}/5\bigr)$ where $\phi_1$ is the first coordinate of the forced-damped Duffing oscillator $$ \ddot\phi + 0.1\,\dot\phi + \phi^3 = 11\cos(\tau). $$ The oscillator is integrated externally (see d_phi.m) and the solution sampled at run time.

MATLAB implementation notes

% Aperiodic forcing setup (case 3)
phiTimespan = [0, 25];
phiInitial  = [0, 0];
phiSol      = ode45(@d_phi, phiTimespan, phiInitial);
phi1        = deval(phiSol, linspace(0,25,1e5), 1);
phi1Max     = max(phi1);

% Pass phiSol and phi1Max through the derivative wrapper
lDerivative = @(t,x,~) derivative(t, x, false, u, lengthX, lengthY, epsilon, 3, phiSol, phi1Max);

Expected output

  • A chain of green elliptic LCSs, one per Rossby-wave cat's eye, lined up along the jet.
  • Red repelling manifolds separating adjacent vortices.
  • A strong mid-channel band of attracting manifolds along the jet core.

Why this demo matters

It is the cleanest published example of elliptic LCSs acting as material vortex boundaries in a perturbed shear flow, and it is the canonical benchmark for geophysical LCS methods.

3. Ocean Dataset real data

Physical setup

A 30-day slice of AVISO surface geostrophic velocities in the Agulhas region (south-west Indian Ocean). The data file ocean_geostrophic_velocity.mat contains:

  • time — vector of days since arbitrary origin,
  • lat, lon — 1-D grids in degrees,
  • vLon, vLat — 3-D arrays of size [nT, nLat, nLon] with velocities in deg/day.

MATLAB implementation

% --- driver ---
load('ocean_geostrophic_velocity.mat')
% Zero-pad boundaries to prevent NaNs leaking
vLon(:,[1,end],:) = 0;  vLon(:,:,[1,end]) = 0;
vLat(:,[1,end],:) = 0;  vLat(:,:,[1,end]) = 0;

vLonInterpolant = griddedInterpolant({time, lat, lon}, vLon, 'spline');
vLatInterpolant = griddedInterpolant({time, lat, lon}, vLat, 'spline');

lDerivative = @(t,x,~) derivative(t, x, vLonInterpolant, vLatInterpolant);
incompressible = true;

The derivative itself is trivially short:

% demo/ocean_dataset/derivative.m
nPosition = numel(position)/2;
derivative_ = nan(nPosition*2, 1);
derivative_(1:2:end-1) = VX_interpolant(time*ones(nPosition,1), position(2:2:end), position(1:2:end-1));
derivative_(2:2:end)   = VY_interpolant(time*ones(nPosition,1), position(2:2:end), position(1:2:end-1));

Key take-aways when adapting to your own data

  1. Use griddedInterpolant({t, y, x}, V, 'spline') — note the axis order t, y, x, not t, x, y.
  2. Zero-pad boundaries of vLon, vLat at indices 1 and end along each spatial axis — stops trajectories at the edge from producing NaN and polluting their neighbours.
  3. Pick the physical resolution the velocity field can support. Oversampling amplifies interpolation noise into $C$.
  4. Use degrees/days consistently throughout (so velocities match position units). For lat/lon at high latitudes, either project to a local flat plane first or add metric factors.
  5. Set eigenvalueFromMainGrid = false — on noisy data, aux-grid eigenvalues behave better.
