Plotting

Every visualisation helper in LCStool — signatures, usage patterns, and colouring conventions.

All plotting helpers are thin wrappers over MATLAB primitives (plot, imagesc, quiver, pcolor). They exist only to (a) honour the correct axis limits / aspect ratio, (b) reshape flattened grid arrays back to 2-D, and (c) return handles that demos customise.

setup_figure

setup_figure plotting

Create an axes with equal aspect ratio and domain-aligned limits.
Signature
hAxes = setup_figure(flowDomain)
Effect
• DataAspectRatio = [1 1 1] — physical aspect is preserved;
• XLim = flowDomain(1,:), YLim = flowDomain(2,:);
• XGrid, YGrid on, Box on, NextPlot = 'add';
• xlabel/ylabel set to "x" / "y".
Use
Always call this before any of the plot_* helpers. Every demo opens its figure this way.

plot_ftle

plot_ftle plotting

Draw an FTLE field as a colour image over the domain.
Signature
[hFtle, hColorbar] = plot_ftle(hAxes, domain, resolution, ftle)
Internals
Reshapes ftle to fliplr(resolution), calls imagesc(domain(1,:), domain(2,:), ftle), and attaches a colorbar.
Common tweaks
% Clip colour range
set(hAxes, 'clim', [0, 0.5]);

% Highlight NaN points (where C was not PD)
ftleField(isnan(ftleField)) = max(ftleField(:));
plot_ftle(hAx, domain, resolution, ftleField);
Math
Plots the field computed by ftle: $\Lambda = \tfrac{1}{2T}\ln\lambda_2$.

plot_elliptic_lcs

plot_elliptic_lcs plotting

Draw the final elliptic LCSs returned by elliptic_lcs.
Signature
h = plot_elliptic_lcs(hAxes, ellipticLcs)
Input
ellipticLcs is a cell array where each cell is an $M\times 2$ closed curve (the outermost-area orbit for one Poincaré section).
Output
h — array of line handles, one per section. Customise colour / linewidth after the call.
Typical usage
hEll = plot_elliptic_lcs(hAx, ellipticLcs);
set(hEll, 'color', [0 .6 0], 'linewidth', 2);
% Keep elliptic boundaries on top of strain lines
uistack(hEll, 'top');

plot_closed_orbit

plot_closed_orbit plotting

Plot all candidate closed orbits at every λ and every section — useful for diagnosis.
Signature
h = plot_closed_orbit(hAxes, closedOrbit)
Input
closedOrbit is the nested cell array returned by poincare_closed_orbit_range: size [nLambda, nPoincareSection], each cell itself a list of orbits for that $(\lambda, \Sigma)$ pair.
Use case
Sanity-check: if many closed orbits appear for a section, the vortex is well-captured. Few or none means either the Poincaré section is misplaced or $\lambda$-sweep is off.

plot_along_arc_length

plot_along_arc_length plotting

Plot a scalar along a curve, parameterised by cumulative arc length.
Signature
hPlot = plot_along_arc_length(hAxes, position, value)
Formula
$$ s(k) = \sum_{j=1}^{k-1} \sqrt{(x_{j+1}-x_j)^2 + (y_{j+1}-y_j)^2},\quad s(1)=0. $$ then plots value against s.
Example use
Plot the eigenvalue $\lambda_2$ sampled along a shrink line to visualise hyperbolicity strength across the curve.

plot_orient_discont

plot_orient_discont plotting

Visualise where the eigenvector field has near-π orientation jumps between neighbours.
Signature
hAxes = plot_orient_discont(eigenvector, domain, resolution) or plot_orient_discont(…, angleTol)
What it does
(a) draws the eigenvector field as a quiver plot;
(b) overlays red markers at cells where the angle between neighbouring eigenvectors exceeds $\pi - \text{angleTol}$ in either $x$- or $y$-direction.
Math
Uses $\theta = \arccos(\mathbf{v}_i \cdot \mathbf{v}_{i+1})$ with $\mathbf{v}_i$ unit-normalised. Cells with $\theta > \pi - \text{angleTol}$ are the suspicious ones.

animate_flow

animate_flow plotting

Show (or record) a particle-cloud animation of the flow — the fastest sanity check you can run.
Signature
flow = animate_flow(flow, animationTime, framerate, animationFilename)
Behaviour
If flow.solution doesn't exist, first runs integrate_flow to build it; then steps through framerate · animationTime + 1 evenly spaced times, evaluating the saved ODE solution with deval at each. If animationFilename is provided, writes frames to a VideoWriter (MP4).
Example
flow.derivative  = @(t,x,eov) derivative(t,x,eov, epsilon, amplitude, omega);
flow.domain      = [0 2; 0 1];
flow.timespan    = [0 20];
flow.resolution  = [20 10];
flow = animate_flow(flow);      % live preview
flow = animate_flow(flow, 10, 20, 'dg.mp4'); % record
Recommended first step
Before computing LCSs, use this to visually confirm your derivative is correct and the particle cloud moves as expected.

Colour convention

All three demos use the same convention, which I recommend adopting:

Curve typeColourWhy
Repelling LCS (shrink line)redHazard / stretching direction
Attracting LCS (stretch line)blueCollects passive tracers
Elliptic LCSgreen [0 .6 0]Vortex boundary / contained region
