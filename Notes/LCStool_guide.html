1. Introduction & theory refresher

LCStool is a MATLAB toolbox developed by G. Haller's group at ETH Zürich (Onu, Huhn & Haller, J. Comput. Sci. 7 (2015) 26–36) for extracting Lagrangian Coherent Structures (LCS) — the most influential material surfaces that organise mixing and transport in a time-dependent 2D flow. The toolbox works entirely on trajectories; it needs only a right-hand side ẋ = v(x,t) and a domain/timespan.

Three families of LCSs are computed:

FamilyPhysical meaningMathematical object
Repelling (hyperbolic) Most repelling material lines — the "spines" of stretching. Shrink lines: integral curves of the minor Cauchy-Green eigenvector ξ1, seeded at local maxima of λ2.
Attracting (hyperbolic) Most attracting material lines. Stretch lines: integral curves of the major eigenvector ξ2, seeded at local minima of λ1.
Elliptic Coherent vortex boundaries. Outermost closed orbits of the generalised ηλ± field (null-geodesics of the Lorentzian metric Eλ = ½(C − λ²I)), detected via Poincaré sections.

All three families are derived from the Cauchy-Green strain tensor C(x0) = ∇FT∇F of the flow map Ft0t. Its eigen-decomposition (λ1, λ2, ξ1, ξ2) with 0 < λ1 ≤ λ2 is the common input to everything downstream.

2. The end-to-end pipeline

Every demo script follows the same five stages. Reading one driver makes the whole toolbox transparent:

┌─────────────────────┐ │ 1. Define flow │ velocity field v(t,x) │ + domain, time, │ domain = [xMin xMax; yMin yMax] │ + resolution │ timespan = [t0 tF] └──────────┬──────────┘ │ ┌──────────▼──────────┐ │ 2. Advect particles │ integrate_flow / eig_cgStrain │ compute C, λ, ξ │ → cgEigenvalue (N×2), cgEigenvector (N×4) └──────────┬──────────┘ │ ┌──────┴──────────────────────────┐ │ │ ┌───▼────────────────┐ ┌───────▼────────────────┐ │ 3a. Elliptic │ │ 3b. Hyperbolic │ │ lambda_line │ │ seed_curves_from_ │ │ → η_±(λ) │ │ lambda_max │ │ poincare_closed_ │ │ → shrink / stretch │ │ orbit_range │ │ lines │ │ elliptic_lcs │ │ remove_strain_in_ │ │ → vortex barriers │ │ elliptic │ └─────────┬──────────┘ └────────────┬────────────┘ │ │ └──────────────┬─────────────────┘ │ ┌──────────▼──────────┐ │ 4. Plot / analyse │ │ plot_elliptic_ │ │ lcs, plot_ftle, │ │ setup_figure … │ └─────────────────────┘

LCStool is procedural and flat: every function is self-contained and configured through name-value pairs via inputParser. There is almost no shared state (the one exception, valueHandle, is used to pass mutable state into ODE event functions).

3. Function-by-function reference

All files live in Codes/LCStool-master/LCStool-1-0/. Tags: util utility · hyperbolic · elliptic · plotting.

3.1 Grid & flow integration

initialize_ic_grid.m util

Build a Cartesian grid of initial conditions.
Signature
position = initialize_ic_grid(resolution, domain)
Returns
position — prod(resolution) × 2 array of (x,y) pairs (meshgrid flattened).
Used by
Every downstream routine that needs uniform seeds.

equal_resolution.m util

Pick resolutionY so that ΔX ≈ ΔY.
Signature
[resolutionY, deltaX] = equal_resolution(domain, resolutionX)
Why needed
The Cauchy-Green calculation (and its auxiliary-grid spacing) assumes square cells. A warning fires if spacing is unequal.

integrate_flow.m util

Advect a cloud of particles with ode45.
Signature
flowSolution = integrate_flow(flow, initialPosition, useEoV, verbose)
Inputs
flow — struct with derivative, timespan, optional odeSolverOptions, coupledIntegration, cgStrainMethod.
useEoV — integrate the 2×2 equation of variation alongside positions (adds 4 extra states per particle).
Modes
Coupled: one huge ode45 call on all particles at once — usually fastest. Uncoupled: one ode45 per particle with parfor.

animate_flow.m plotting

Quick sanity-check animation of the particle cloud.
Signature
flow = animate_flow(flow, animationTime, framerate, animationFilename)
Notes
Writes an MP4 (via VideoWriter) if animationFilename is given. Integrates the flow first if flow.solution is missing.

apply_periodic_bc.m util

Wrap positions across periodic axes (currently only the x-direction is implemented).

3.2 Cauchy-Green strain & FTLE

eig_cgStrain.m util

The workhorse. Computes eigenvalues/eigenvectors of C on the whole grid.
Signature
[cgV, cgD] = eig_cgStrain(derivative, domain, resolution, timespan, ...)
Key options
'method' : 'finiteDifference' (default) or 'equationOfVariation'.
'auxGridRelDelta' (default 1e-2) — spacing of auxiliary grid used to compute ∇F.
'eigenvalueFromMainGrid' (default true) — compute eigenvalues on main grid, eigenvectors on auxiliary.
'incompressible' — enforce λ1·λ2=1 by setting λ1 = 1/λ2.
'odeSolverOptions' — odeset struct.
Outputs
cgD : N×2 eigenvalues [λ1, λ2].
cgV : N×4 eigenvectors packed as [ξ1x, ξ1y, ξ2x, ξ2y].
Internals
Calls ode45_vector (private), auxiliary_position, compute_cgStrain, and eig_array; points where C is not positive-definite are set to NaN by negative_to_nan.

ftle.m util

Finite-Time Lyapunov Exponent from largest Cauchy-Green eigenvalue.
Formula
ftle = 0.5·log(λ2) / T

cgStrain_stats.m / eig_error.m util

Diagnostics: report NaN counts, residual of the eigenvalue problem, deviation of det(C) from 1 (a check on incompressibility).

3.3 Hyperbolic LCS — shrink & stretch lines

seed_curves_from_lambda_max.m hyperbolic

Place curves at local extrema of an eigenvalue field and integrate the corresponding eigenvector line.
Signature
[curves, seeds] = seed_curves_from_lambda_max(distance, maxLen, cgEig, cgVec, domain, resolution, ...)
Arguments
distance — minimum separation between two seeds, prevents curve overlap.
maxLen — max arc length integrated (forward + backward).
cgEig — pass cgD(:,2) for shrink, -cgD(:,1) for stretch.
cgVec — corresponding eigenvector columns cgV(:,1:2) or cgV(:,3:4).
Optional 'periodicBc', 'nMaxCurves', 'odeSolverOptions'.
Algorithm
(1) find local maxima on a grid with circular-mask distance threshold; (2) sort by value; (3) from each seed integrate the eigenvector field forward and backward with integrate_line; (4) flag grid cells touched, skip overlapping seeds.

integrate_line.m util

Generic integrator for non-orientable vector fields (used by every strain-line and λ-line integration).
Key tricks
• Builds a griddedInterpolant for the vector field.
• Uses a valueHandle to remember the previous direction; flips sign when the dot-product goes negative, so the integration follows the line-field continuously.
• Detects orientation discontinuities across cells (angle between neighbours close to π/2) and replaces the local interpolant with a "continuous" one.
• Supports event detection: stop at domain boundary, at NaN, or at a Poincaré section crossing.

remove_strain_in_elliptic.m hyperbolic

Trim shrink/stretch lines that cross an elliptic LCS boundary (the vortex interior is excluded from hyperbolic analysis).
Caveat (flagged in source)
Assumes each strainline crosses an elliptic region at most once; violating this discards legitimate outside segments.

angle_change.m util / plot_orient_discont.m plotting

Quantify and visualise orientation discontinuities of the eigenvector field.

3.4 Elliptic LCS — λ-lines & Poincaré closed orbits

lambda_line.m elliptic

Build the η+λ and η−λ vector fields (null-geodesics of the Lorentzian metric), eq. 3.2 of DOI:10.1017/jfm.2013.391.
Formula
η± = √((λ2 − λ²)/(λ2 − λ1))·ξ1 ± √((λ² − λ1)/(λ2 − λ1))·ξ2
Option
'forceComplexNaN' replaces locations where the expression is complex (when λ lies outside [λ1, λ2]) with NaN.

poincare_closed_orbit.m elliptic

For a single Poincaré section and one η-field, find closed orbits and select the outermost.
Algorithm
(1) seed numPoints particles along the section; (2) integrate each with integrate_line until it returns to the section; (3) compute signed return distance; (4) find zero-crossings of "return − initial"; (5) refine each with bisection (nBisection iterations); (6) keep zero-crossings whose neighbours also return (small dThresh); (7) sort by distance from section start (innermost → outermost).

poincare_closed_orbit_multi.m elliptic

Loop poincare_closed_orbit over a list of Poincaré sections and over both η+ and η−.

poincare_closed_orbit_range.m elliptic

The outer loop: sweep a range of λ values; call lambda_line and poincare_closed_orbit_multi for each.
Outputs
closedLambdaLinePos, closedLambdaLineNeg — cell arrays of size [nLambda, nPoincareSection].

elliptic_lcs.m elliptic

Final selection: for each Poincaré section keep the outermost closed λ-line with the largest enclosed area (via polyarea). Sections with no closed orbit are dropped.

3.5 Utilities, plotting, diagnostics

valueHandle.m util

One-property handle class. The only OO construct in the toolbox — a trick to pass mutable state (e.g. last vector direction) into ODE output/event functions.

setup_figure.m plotting

Create a figure with equal axes and domain limits — used at the top of every demo.

plot_ftle.m plotting

pcolor of an FTLE field reshaped to the grid.

plot_elliptic_lcs.m / plot_closed_orbit.m / plot_along_arc_length.m plotting

Convenience wrappers for drawing elliptic boundaries, candidate closed orbits, and scalar fields parameterised along a curve.

print_theta_hist.m plotting

Histogram of angle changes between eigenvectors of neighbouring grid cells — a quick QA check on the smoothness of the eigenvector field.

4. The three demo cases

Each demo folder ships three files of interest: derivative.m (the velocity RHS), elliptic_hyperbolic_lcs.m (the driver script), and flow_animation.m (a visual sanity check). The structure of all three drivers is identical — only the velocity definition and parameter values change.

4.1 Double Gyre textbook

What it is. The classic time-periodic Shadden double-gyre (DOI:10.1016/j.physd.2005.10.007): two counter-rotating cells on [0,2]×[0,1] with a time-dependent dividing line.

Velocity (see demo/double_gyre/derivative.m):

% parameters: epsilon=0.1, amplitude=0.1, omega=pi/5
a = epsilon*sin(omega*t);
b = 1 - 2*epsilon*sin(omega*t);
f = a*x.^2 + b*x;
u = -pi*A*sin(pi*f)*cos(pi*y);
v =  pi*A*cos(pi*f)*sin(pi*y)*(2*a*x + b);

Key parameters in the driver:

ParameterValueMeaning
timespan[0, 10]advection window
domain[0,2; 0,1]physical domain
resolutionX500main grid x-resolution
lambda0.93 : 0.01 : 1.07λ-sweep for elliptic LCS
poincareSection(1).endPosition[.55,.55 ; .1,.1]left vortex
poincareSection(2).endPosition[1.53,.45 ; 1.95,.05]right vortex
shrinkLineMaxLength20arc-length cap for repelling LCSs

Expected output. One green elliptic LCS per vortex, plus dense red/blue "spines" of repelling and attracting manifolds that sweep around the gyres.

4.2 Bickley Jet geophysical

What it is. An idealised zonal jet with three travelling Rossby-wave perturbations on a periodic zonal channel — a standard benchmark for geophysical LCS (Beron-Vera et al., DOI:10.1016/j.physd.2012.06.012).

Velocity structure. The stream function is

ψ = U·L·(tanh(y/L) − sech²(y/L)·Re[Σn εn·fn(t)·exp(i kn x)]) − c3·y

which gives the u,v returned by demo/bickley_jet/derivative.m. Three perturbation cases are available:

  • case 1 — time-periodic ψ1.
  • case 2,3 — time-aperiodic forcing driven by an auxiliary ODE (d_phi.m) integrated beforehand.

Characteristic setup:

  • domain = [0, π·REarth; −2.25·LY, 2.25·LY]
  • periodicBc = [true, false] — x-periodicity is essential.
  • Five Poincaré sections — one for each of the expected coherent vortices along the jet (the "Rossby-wave rings").
  • shrinkLineMaxLength = 1e8 m — much longer than the channel because strainlines can wrap many times.

Why it matters. It's the cleanest published example of elliptic LCS acting as material vortex boundaries in a perturbed shear flow.

4.3 Ocean Dataset real data

What it is. A 30-day slice of AVISO geostrophic surface velocities in the Agulhas region (south-west Indian Ocean). Data is shipped as ocean_geostrophic_velocity.mat with the variables time, lat, lon, vLon, vLat.

Velocity is supplied by bilinear/spline interpolation (demo/ocean_dataset/derivative.m):

vLonInterpolant = griddedInterpolant({time, lat, lon}, vLon, 'spline');
vLatInterpolant = griddedInterpolant({time, lat, lon}, vLat, 'spline');
lDerivative = @(t,x,~) derivative(t, x, vLonInterpolant, vLatInterpolant);

Setup:

  • timespan = [100, 130] (days since arbitrary origin).
  • domain = [0°, 6° lon; −34°, −28° lat].
  • 5 Poincaré sections placed by hand over suspected mesoscale eddies.
  • cgEigenvalueFromMainGrid = false — eigenvalues from the auxiliary grid are more accurate for noisy data.
  • λ-sweep 0.9 : 0.02 : 1.1, slightly coarser than the analytical demos.

Takeaway for users of real data. The only differences from the analytical demos are: (a) derivative uses griddedInterpolant, (b) you must zero-pad the velocity at the boundaries to avoid NaNs leaking into trajectories, (c) degrees/days are used consistently everywhere (so speeds are in deg/day).

5. How to use LCStool on your own velocity field

Two scenarios cover almost every case.

5.1 Analytical velocity (you have equations)

Write a derivative.m that matches LCStool's convention and copy a demo driver. The convention is strict:

  • position comes in as a column vector stacked as [x1;y1;x2;y2;…;xN;yN];
  • Your function must return a column of the same length with [u1;v1;u2;v2;…;uN;vN];
  • A third argument useEoV is required; if you don't support the equation of variation, ignore it (set useEoV=false always when wrapping).

Minimal template:

function dydt = my_derivative(t, x, useEoV)
    idxX = 1:2:numel(x)-1;
    idxY = 2:2:numel(x);
    dydt = nan(size(x));
    % --- your velocity ---
    dydt(idxX) =  sin(x(idxY));
    dydt(idxY) = -cos(x(idxX));
end

Then the driver:

addpath('path/to/LCStool-1-0');

%% 1. Domain, grid, time
domain      = [0, 2*pi; 0, 2*pi];
timespan    = [0, 20];
resolutionX = 400;
[resolutionY, deltaX] = equal_resolution(domain, resolutionX);
resolution  = [resolutionX, resolutionY];

%% 2. Velocity wrapper
lDerivative   = @(t,x,~) my_derivative(t, x, false);
incompressible = true;

%% 3. Cauchy-Green
cgOde = odeset('relTol', 1e-5);
[cgV, cgD] = eig_cgStrain(lDerivative, domain, resolution, timespan, ...
                'incompressible', incompressible, ...
                'odeSolverOptions', cgOde);

%% 4. Hyperbolic LCS: shrink (repelling) & stretch (attracting) lines
shrinkOde = odeset('relTol', 1e-6);
shrinkLine = seed_curves_from_lambda_max(2*deltaX, 20, cgD(:,2), cgV(:,1:2), ...
                                        domain, resolution, ...
                                        'odeSolverOptions', shrinkOde);
stretchLine = seed_curves_from_lambda_max(10*deltaX, 20, -cgD(:,1), cgV(:,3:4), ...
                                        domain, resolution, ...
                                        'odeSolverOptions', shrinkOde);

%% 5. Elliptic LCS: choose Poincaré sections through candidate vortex cores
poincareSection = struct('endPosition',{},'numPoints',{},'orbitMaxLength',{});
poincareSection(1).endPosition = [pi, pi; pi+1, pi+1];  % tailor to your flow
[poincareSection.numPoints] = deal(100);
for i = 1:numel(poincareSection)
    r = hypot(diff(poincareSection(i).endPosition(:,1)), ...
              diff(poincareSection(i).endPosition(:,2)));
    poincareSection(i).orbitMaxLength = 2*(2*pi*r);
end
lambda = 0.9 : 0.02 : 1.1;

[clPos, clNeg] = poincare_closed_orbit_range(domain, resolution, cgV, cgD, ...
                    lambda, poincareSection, 'forceEtaComplexNaN', true);
ellipticLcs = [elliptic_lcs(clPos), elliptic_lcs(clNeg)];

%% 6. Plot
hAx = setup_figure(domain);
plot_elliptic_lcs(hAx, ellipticLcs);
cellfun(@(p) plot(hAx, p(:,1), p(:,2), 'r'), shrinkLine);
cellfun(@(p) plot(hAx, p(:,1), p(:,2), 'b'), stretchLine);
Tip. If you want just an FTLE map (no LCSs), you only need eig_cgStrain → ftle → plot_ftle. That's three lines.

5.2 Gridded velocity data (CFD output, satellite, model)

Adapt the ocean demo. The only thing to change is how derivative is built:

% Suppose you have: t (nT×1), x (nX×1), y (nY×1),
%                   U, V both size [nT × nY × nX] (note Y before X for griddedInterpolant)

U_int = griddedInterpolant({t, y, x}, U, 'spline');
V_int = griddedInterpolant({t, y, x}, V, 'spline');

function dydt = my_deriv_data(t, pos, Uin, Vin)
    N = numel(pos)/2;
    dydt = nan(2*N, 1);
    xPart = pos(1:2:end-1);
    yPart = pos(2:2:end);
    tCol  = t*ones(N,1);
    dydt(1:2:end-1) = Uin(tCol, yPart, xPart);
    dydt(2:2:end)   = Vin(tCol, yPart, xPart);
end

lDerivative = @(t,x,~) my_deriv_data(t, x, U_int, V_int);

Then copy the rest of the ocean driver. Remember:

  • Zero the velocity in the first and last index of each dimension, otherwise trajectories hitting the edge return NaN and pollute neighbours.
  • Use a spatial resolution the velocity data can support — oversampling just amplifies interpolation noise into the Cauchy-Green field.
  • For curvilinear or lat/lon data, either convert to a local flat projection or include the metric factors in your derivative.

5.3 A practical order of operations

  1. First run only integrate_flow + animate_flow — verify the particles move sensibly.
  2. Second compute eig_cgStrain and plot FTLE = 0.5·log(λ2)/T with plot_ftle. Ridges in FTLE pre-visualise the hyperbolic LCS locations; if they look sensible, your pipeline is wired correctly.
  3. Third run seed_curves_from_lambda_max.
  4. Last pick Poincaré sections by eye (through clear FTLE eddy cores) and run poincare_closed_orbit_range. This step is the fussiest — expect to iterate on section placement and the dThresh/nBisection parameters.

6. Common pitfalls & tuning tips

MATLAB version. seed_curves_from_lambda_max hard-errors on releases older than R2013a ("silent failures" were observed on R2011b).
Axis order. LCStool stores positions as [x y] rows, but reshapes use fliplr(resolution) because MATLAB arrays are row-major with y along the first dimension. Whenever you write a new plotting helper, double-check that the reshape matches what griddedInterpolant({y, x}, …) expects.
NaNs in the eigen-fields. The toolbox produces NaN on three occasions: (1) when C loses positive-definiteness (numerical error), (2) when a particle leaves the domain, (3) when 'incompressible' pushes λ1 ≥ 1. Downstream routines tolerate NaNs, but a large NaN fraction indicates integration tolerances are too loose.
λ-sweep granularity. For analytical flows 0.01 step is usually fine; for noisy ocean data 0.02–0.05 is enough and much faster.
Poincaré section placement. A section should cross the suspected vortex once and be longer than the largest expected orbit diameter. Place endPosition(1,:) near the eddy centre and endPosition(2,:) outside its edge.
Parallelism. integrate_flow and poincare_closed_orbit use parfor. Start a parallel pool (parpool) before calling them for large grids.
"Large angle discontinuity" warning. This flags cells where neighbouring eigenvector orientations differ by nearly π/2. Small counts are harmless; clustered discontinuities suggest the auxiliary-grid spacing (auxGridRelDelta) is too small or the underlying velocity field is under-resolved.
